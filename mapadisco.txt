El disco de la abadía del crimen tiene 0x28 pistas (1 cara por pista). Los datos están guardados al revés (el último byte es el primero, 
el penúltimo byte es el segundo, etc)

Todas las pistas menos la 0, tiene 15 sectores de 0x0100 bytes, por lo que cada pista contiene 0x0f00 bytes
La distribución de los datos en las pistas es la siguiente:
0x00: pista de arranque llamada al escribir |CPM -> longitud = 0x0300 bytes
0x01-0x11: pistas que tienen los datos de abadia0.bin, abadia3.bin, abadia8.bin y abadia1.bin -> longitud = 0xff00 bytes
0x12-0x16: pistas que tienen los datos de abadia7.bin y sobreescriben los últimos 0xb00 bytes de abadia1.bin -> longitud = 0x4b00 bytes
0x17-0x1b: pistas que tienen los datos de abadia6.bin y sobreescriben los últimos 0xb00 bytes de abadia1.bin -> longitud = 0x4b00 bytes
0x1c-0x21: pistas que tienen los datos de abadia5.bin y sobreescriben los últimos 0xb00 bytes de abadia1.bin -> longitud = 0x4b00 bytes
0x21-0x25: pistas que tienen los datos de abadia2.bin y sobreescriben los últimos 0xb00 bytes de abadia1.bin -> longitud = 0x4b00 bytes
